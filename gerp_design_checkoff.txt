Answer:

Let k be the number of matching lines for a query, and let L be the total number of 
characters we print for those matching lines. We plan to use a hash table based index,

8. How will you handle case-sensitive search?
   What is the time complexity (should be close to O(1))?

Answer:

For case-sensitive search, we will maintain an index that stores each processed word exactly as it 
appears after stripping leading and trailing non-alphanumeric characters. When the user enters a normal 
query, we will strip leading and trailing non-alphanumeric characters from that query and then look it 
up directly in the case-sensitive index. The lookup itself should be close to O(1) on average and then 
printing the results takes time proportional to the number of matching lines. 

9. How will you handle case-insensitive search?
   What is the time complexity (should be close to O(1))?

Answer:

For case-insensitive search we will build a second index. In that index, every processed word is converted 
to lowercase before being stored. So we, We, and WE would all map to the same key. Then when the user uses 
@i, we will strip the query, convert it to lowercase, and do a direct lookup in the case-insensitive index. 
That lookup should also be close to O(1) on average, followed by the time needed to print the matching lines. 
We are using two separate indexes because it keeps the search logic much simpler and faster.

10. How are you handling duplicate words within a line? Consider these two example 
   lines: 

      (i) the cat and the dog 
      (ii) the cat and THE dog 

   If a user does a case sensitive query on line (i) it should report 'the' only 
   once by showing the line only once in the output. How will you do this? 

   If a user does a case insensitive query on line (ii) it should report 'the' 
   only once by showing the line only once in the output. How will you do this? 

Answer:

We only want each line to appear once in the output, even if the word shows up multiple times in that line. 
To do this, when we process a line, we keep track of the words we’ve already seen using a temporary set. 
For case-sensitive search, we store each cleaned word once per line, so duplicates like “the” don’t add the 
same line multiple times. For case-insensitive search, we do the same thing but convert words to lowercase 
first, so “the” and “THE” are treated as the same word and still only add the line once.
